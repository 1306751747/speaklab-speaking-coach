# 评审提交清单

用于提交前自检，确保仓库、PR、README 和 Demo 视频符合比赛要求。

## 仓库

- 仓库需在提交截止后公开可访问。
- 主分支代码保持可运行。
- README 中列明依赖、运行方式、原创功能范围和 Demo 视频链接。
- 不提交 `outputs/`、`work/` 等本地生成目录。

## PR

- 每个 PR 只做一件事。
- PR 标题用一句话说明变更。
- PR 描述包含功能描述、实现思路和测试方式。
- 如复用历史代码片段，需要在 PR 描述中注明来源。

## Demo 视频

- 视频需包含声音讲解。
- 展示场景选择、语音对话、即时纠错、评分和课后总结。
- 链接需放入 README，并确保评委可访问播放。

## 发布前验证

```bash
node --check app/app.js
git status --short --branch
```
