## 2. 一句话摘要

SpeakLab 是一款面向中文学习者的 AI 英语口语练习工具。它用温暖低压力的视觉体验、任务化口语场景、自由英语对话、中文表达脚手架和温柔 AI 反馈，帮助用户更稳定地开口练习，并把每次练习沉淀为可复习的表达资产。

## 4. 需求背景

### 4.1 用户痛点

中文学习者在练英语口语时，常见问题不是单纯“不知道语法”，而是：

- 不敢开口，害怕说错。
- 不知道今天该练什么。
- 有中文想法，但不知道怎么组织成英文。
- 练完以后不知道哪里说得好、哪里要改。
- 纠错太严厉时容易产生挫败感。
- AI 对话如果没有任务边界，用户容易卡住。
- 练习记录如果不能沉淀，用户很难看到长期进步。

### 4.2 市场背景

市面上的 AI 英语口语产品已经具备对话、场景、评分、纠错、课后报告等能力。主流竞品的共同趋势是：

- 用一个强入口降低用户启动成本。
- 用场景或角色扮演降低表达难度。
- 用即时反馈增强练习闭环。
- 用学习进度和总结提升复访。

SpeakLab 的差异化不应该只做“AI 陪练 + 打分”，而应该聚焦中文学习者更真实的需求：低压力开口、中文引导、温柔纠错、表达复用和持续成长。

### 4.3 为什么现在做

大模型已经可以生成自然对话、结构化反馈和多版本英文表达。浏览器也提供 Web Speech API，可以低成本实现语音输入。因此，一个轻量 Web 产品已经可以完成较完整的口语练习闭环。

## 5. 产品目标

### 5.1 产品愿景

让中文学习者在温暖、低压力、可持续的环境里练习真实英语表达。

### 5.2 阶段目标

| 目标 | 说明 |
| --- | --- |
| 降低开口压力 | 用治愈系视觉、中文界面和温柔纠错减少焦虑 |
| 降低选择成本 | 用今日任务、场景分类和自由对话入口帮助用户快速开始 |
| 提高练习有效性 | 每个场景提供任务、必说点和可用句型 |
| 提高反馈价值 | 反馈不只打分，还给原句、改写、原因和下次重点 |
| 建立复习闭环 | 将推荐改写、常错表达和高分句型保存为表达资产 |

### 5.3 成功指标

| 指标 | 目标值 | 说明 |
| --- | --- | --- |
| 首屏进入练习转化率 | >= 60% | 访问首页后点击进入练习工作台的比例 |
| 单次练习完成率 | >= 50% | 进入练习页后提交至少一次回答的比例 |
| 自由对话使用率 | >= 20% | 用户选择自由对话模式的比例 |
| 场景切换使用率 | >= 30% | 用户主动切换练习场景的比例 |
| 课后总结曝光率 | >= 80% | 提交回答后看到总结卡的比例 |
| 表达资产生成率 | >= 50% | 每次有效练习后沉淀表达资产的比例 |
| 用户主观压力评分 | 低于传统纠错型产品 | 通过访谈或问卷验证 |

## 6. 目标用户

### 6.1 核心用户

| 用户类型 | 典型特征 | 核心需求 |
| --- | --- | --- |
| 初中级英语学习者 | 有词汇基础，但不敢说完整句 | 希望低压力开口，并得到中文解释 |
| 求职/职场用户 | 需要面试、会议、汇报、项目沟通 | 希望练真实场景表达 |
| 出行用户 | 需要旅行、酒店、机场、问路等表达 | 希望短时间掌握实用句 |
| 自我提升型用户 | 希望每天练一点口语 | 希望看到成长和复盘 |

### 6.2 非核心用户

- 高级口译或专业发音训练用户。
- 需要真人外教实时视频互动的用户。
- 只想背单词或刷题的用户。

## 7. 名词解释

| 名词 | 解释 |

| 互动首页 | 用户进入产品后看到的 warm clay 语言花园页面 |

| 温柔纠错 | 先肯定表达，再指出一个关键问题，最后给推荐表达的反馈方式 |

## 9. 用户路径

### 9.1 首次访问路径

```text
打开公网地址
-> 看到互动首页
-> 移动鼠标体验玻璃圆圈和单词溶解
-> 点击 Start Communicating
-> 进入中文口语练习工作台
-> 查看今日任务
-> 选择场景或自由对话
-> 输入或语音回答
-> 查看实时反馈和课后总结
-> 保存表达资产
```

### 9.2 场景练习路径

```text
进入练习工作台
-> 选择“场景练习”
-> 选择场景分类
-> 点击具体场景
-> 阅读任务、必说点、可用句型
-> 输入英文回答或点击麦克风
-> 提交回答
-> 查看评分、纠错、推荐改写、下一句追问
-> 生成课后总结和表达资产
```

### 9.3 自由对话路径

```text
进入练习工作台
-> 点击自由对话
-> 输入任意想聊的话题
-> AI 用简短英文自然接话
-> AI 适当追问
-> 用户继续回答
-> 系统给轻量反馈
```

### 9.4 中文表达脚手架路径

```text
用户有中文想法
-> 输入“我想表达...”
-> 点击生成英文表达
-> 获得简单版、自然版、正式版
-> 选择其中一句放入回答框
-> 继续口语练习
```

### 10.2 首页需求

#### 需求 H1：温暖粘土语言花园

说明：

- 页面背景使用 warm clay yellow。
- 有可见的手工粘土质感。
- 小花、小草、小叶子作为轻量装饰。
- 整体氛围温暖、可爱、低压力。

验收标准：

- 用户首屏能感知到粘土花园风格。
- 页面不应像普通黄色平面或商务后台。
- 装饰元素不遮挡主按钮。

#### 需求 H2：英文单词漂浮

说明：

- 白色英文单词随机出现在页面各处。
- 单词缓慢出现、漂浮、淡出。
- 不能从中心向外扩散。

验收标准：

- 页面上持续有英文单词出现。
- 动画不明显卡顿。
- 单词不影响主按钮阅读。

#### 需求 H3：鼠标玻璃圆圈

说明：

- 鼠标位置有一个 iOS frosted glass 风格圆圈。
- 圆圈中心需要和真实鼠标位置对齐。
- 圆圈只影响单词，不影响背景和花草。

验收标准：

- 圆圈跟随没有明显偏移。
- 单词进入圆圈后有自然消失或溶解效果。
- 背景不会被擦除。

### 10.3 练习工作台需求

#### 需求 W1：信息架构

说明：

- 顶部展示欢迎语、今日任务、肯定语和统计。
- 左侧展示练习模式和场景选择。
- 右侧展示主练习区和反馈区。

验收标准：

- 用户进入页面后能快速知道今天练什么。
- 用户能找到在哪里选择场景。
- 用户能找到在哪里输入或点击麦克风。

#### 需求 W2：中文界面

说明：

- UI 文案必须以中文为主。
- 英文只出现在练习内容、句型和 AI 回复中。

验收标准：

- 不出现大面积英文营销标题。
- 模块标题、按钮、提示语均为中文。

#### 需求 W3：视觉统一

说明：

- 练习页延续首页的 warm clay / soft glass 视觉系统。
- 练习页颜色比首页更浅，更适合长时间阅读。

验收标准：

- 用户从首页进入练习页时不会觉得跳到另一个产品。
- 卡片、按钮、滚动条、输入框风格统一。

### 10.4 场景练习需求

#### 需求 S1：场景分类

分类：

- 基础生活。
- 学习工作。
- 社交表达。
- 旅行应急。

验收标准：

- 分类默认不应一次性展开所有场景造成拥挤。
- 当前分类下的场景可点击切换。

#### 需求 S2：场景任务内容

每个场景需要包含：

- 标题。
- 副标题或目标。
- 任务说明。
- 必说点。
- 可用句型。
- 教练追问。

验收标准：

- 切换场景后主练习区内容同步变化。
- 不存在空标题、空任务或空句型。

### 10.5 自由对话需求

说明：

- 自由对话是独立模式，不属于某个固定场景。
- 用户可以输入任何想聊的话题。
- AI 回复要短、自然，像真实聊天。

验收标准：

- 点击自由对话后，标题切换为“自由对话”。
- 输入框 placeholder 切换为自由对话提示。
- 可用句型切换为自由聊天句型。
- AI 反馈更偏自然度和继续表达，不应像作文批改。

### 10.6 输入与语音需求

说明：

- 支持文本输入。
- 支持浏览器麦克风输入。
- 支持提交回答。
- 支持清晰的空状态。

验收标准：

- 输入英文回答后可以提交。
- 麦克风权限正常时可以识别语音。
- 未输入内容时不应产生无效反馈。
- 语音不可用时需要有可理解的提示或回退到文本输入。

### 10.7 AI 反馈需求

#### `/api/feedback`

输入：

- 用户英文回答。
- 当前场景或自由对话模式。
- 本地规则初评分。

输出：

- 综合分。
- 流利度。
- 语法。
- 场景相关度或自然度。
- 发音占位评分。
- 纠错建议。
- 推荐改写。
- 下一句追问。
- 课后总结。
- 温柔纠错结构。

验收标准：

- API Key 存在时优先请求 AI。
- API Key 不存在或请求失败时走本地兜底。
- 前端不会暴露 API Key。
- 输出结构稳定，前端不会因为字段缺失崩溃。

### 10.8 中文表达脚手架需求

#### `/api/scaffold`

输入：

- 用户中文想法。
- 当前场景信息。

输出：

- 简单版英文。
- 自然版英文。
- 正式版英文。
- 中文练习提示。

验收标准：

- 用户可以输入中文想法。
- 系统生成三个英文版本。
- 用户可以选择一个版本放入回答框。
- API 不可用时使用本地模板兜底。

### 10.9 表达资产库需求

说明：

- 每次练习后保存有价值的表达。
- 保存内容包括推荐改写、常错表达和高分句型。
- 当前使用浏览器 `localStorage`。

验收标准：

- 提交回答后表达资产数量可能增加。
- 刷新页面后资产仍可保留。
- 不应无限增长导致页面难以阅读。

## 11. 非功能需求

### 11.1 性能

- 首页动画应尽量使用 `transform` 和 `opacity`。
- 点击进入练习页时应停止不必要动画。
- 转场期间不应逐个操作大量单词。
- 首屏交互应尽量流畅。

### 11.2 可用性

- 用户不需要阅读说明就能开始练习。
- 页面在桌面和移动端都能使用。
- 输入区域、场景区域和反馈区域应可滚动。
- 滚动条视觉需要融入 UI，不使用突兀系统默认样式。

### 11.3 兼容性

- 推荐 Chrome 或 Edge。
- Web Speech API 在不同浏览器支持程度不同。
- 没有麦克风权限时，文本输入仍可使用。

### 11.4 安全与隐私

- API Key 只放在服务端环境变量。
- 不提交 `.env.local`。
- 当前表达资产存储在用户浏览器本地。
- 当前没有用户账号和云端个人数据。

### 11.5 可部署性

- 支持本地开发。
- 支持 Netlify 公网部署。
- 需要 `netlify.toml` 和 `@netlify/plugin-nextjs` 支持 Next.js runtime。

## 12. 技术方案

### 12.1 技术栈

### 12.2 API 设计

#### `/api/feedback`

用途：

生成口语练习反馈。

核心字段：

```json
{
  "answer": "I handled the project timeline.",
  "scenario": {
    "id": "interview",
    "mode": "scenario",
    "label": "面试",
    "title": "面试练习",
    "prompt": "请试着用英语说说你最近的进步和变化。",
    "goal": "练习清晰表达经历和成果。"
  },
  "localAnalysis": {
    "overall": 76
  }
}
```

#### `/api/scaffold`

用途：

把中文想法转换为英文表达。

核心字段：

```json
{
  "idea": "我想说自己负责了项目排期。",
  "scenario": {
    "label": "会议",
    "title": "会议练习"
  }
}
```

## 16. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Web Speech API 浏览器兼容不一致 | 语音输入不稳定 | 保留文本输入作为主路径 |
| AI API 未配置或失败 | 反馈质量下降 | 使用本地规则兜底 |
| 用户不知道先练什么 | 启动率下降 | 强化今日任务和默认场景 |
| 反馈太严厉 | 用户不愿继续练 | 使用温柔纠错结构 |
| 场景过多导致界面拥挤 | 理解成本上升 | 使用分类和收纳式展示 |
| Netlify 环境变量未配置 | 公网 AI 不可用 | README 明确配置方式 |
| localStorage 数据仅本地可用 | 无法跨设备复习 | 后续版本引入登录和云端同步 |

## 17. 依赖与约束

### 17.1 外部依赖

- OpenAI-compatible API。
- Netlify。
- Browser Web Speech API。
- 用户浏览器麦克风权限。

### 17.2 设计约束

- 首页和练习页必须属于同一视觉系统。
- UI 语言必须以中文为主。
- 视觉风格不能变成冷色科技风或普通后台系统。
- 动效要温柔，不能影响核心练习效率。

### 17.3 技术约束

- 当前没有数据库。
- 当前没有账户体系。
- 当前发音评分不是音素级专业评测。
- 当前 AI 回复不是实时语音流。

## 19. 评审展示建议

推荐展示顺序：

1. 打开公网首页。
2. 展示 clay 语言花园和鼠标玻璃圆圈。
3. 点击进入练习工作台。
4. 展示中文欢迎区和今日任务。
5. 切换“面试”或“旅行”场景。
6. 展示任务、必说点和可用句型。
7. 输入英文回答并提交。
8. 展示评分、温柔纠错和课后总结。
9. 使用中文表达脚手架。
10. 切换自由对话，展示自然追问。
11. 展示表达资产库。

## 20. 结论

SpeakLab 的核心不是“再做一个 AI 口语打分器”，而是做一个适合中文学习者的低压力口语成长工具。当前版本已经完成从首页吸引、场景进入、开口练习、AI 反馈、课后总结到表达沉淀的基础闭环。后续版本应继续强化长期成长数据、语音评测能力和多轮真实对话体验。
